## Why This Matters

### Current Problem
```javascript
// ‚ùå Frontend doing database operations
const res = await supaFetch('expenses', {
  method: 'POST',
  body: expenseData
}, companyId);
```

**Issues:**
- ‚ùå RLS policy blocks with 403 Forbidden
- ‚ùå No validation at database level
- ‚ùå No audit trail
- ‚ùå No approval workflow enforcement
- ‚ùå Doesn't work on mobile/desktop apps

### Solution
```javascript
// ‚úÖ Backend RPC handles all logic
const res = await supaFetch('rpc/create_expense', {
  method: 'POST',
  body: {
    p_company_id: companyId,
    p_amount: expenseData.amount,
    p_category_id: expenseData.category_id,
    // ... validation happens in database
  }
}, companyId);
```

**Benefits:**
- ‚úÖ No 403 errors (SECURITY DEFINER)
- ‚úÖ Validation at database level
- ‚úÖ Audit trail created
- ‚úÖ Approval workflows enforced
- ‚úÖ Works on web, mobile, desktop
