## üéØ Industry Standard Pipeline (Quote-to-Cash)

### **Phase 1: Lead/Request**
**Status:** `draft` or `lead`
**Tables:** `work_orders`, `customers`, `service_addresses`

**Actions:**
- Customer requests service (phone, web form, portal, marketplace)
- Office creates draft quote/estimate
- Captures customer info, service address, problem description

**Pain Points Found:**
- ‚ùå Lost leads due to slow response time
- ‚ùå No automated lead assignment
- ‚ùå Can't track lead source/conversion

---

### **Phase 2: Quote/Estimate Creation**
**Status:** `quote` or `draft`
**Tables:** `work_orders`, `work_order_items`, `quote_templates`

**Actions:**
- Create line items (labor, materials, services)
- Apply pricing (flat rate, T&M, unit-based, percentage, recurring)
- Calculate taxes, discounts
- Add photos, diagrams, scope of work
- Use templates for common jobs

**Pain Points Found:**
- ‚ùå **Slow quote creation** - Takes too long to build quotes
- ‚ùå **No templates** - Have to rebuild same quotes repeatedly
- ‚ùå **Poor mobile experience** - Can't create quotes in field easily
- ‚ùå **No optional line items** - Can't let customer choose add-ons
- ‚ùå **Pricing errors** - Manual calculation mistakes

**Industry Best Practices:**
- ‚úÖ Quote templates by job type
- ‚úÖ Optional line items (customer can select)
- ‚úÖ Multiple pricing models in one quote
- ‚úÖ Photo/video attachments
- ‚úÖ Digital signature capture
- ‚úÖ Expiration dates (30/60/90 days)

---

### **Phase 3: Quote Delivery**
**Status:** `sent` or `pending`
**Tables:** `work_orders`, `quote_deliveries`, `documents`, `messages`

**Actions:**
- Generate PDF
- Send via email + portal link
- Optional SMS notification
- Track delivery (sent, viewed, opened)

**Pain Points Found:**
- ‚ùå **No delivery tracking** - Don't know if customer viewed quote
- ‚ùå **Email gets lost** - Customers say they never received it
- ‚ùå **No follow-up reminders** - Quotes go stale
- ‚ùå **Can't resend easily** - Have to regenerate and resend manually
- ‚ùå **No portal access** - Customer has to dig through email

**Industry Best Practices:**
- ‚úÖ Multi-channel delivery (email, SMS, portal)
- ‚úÖ Track quote views (timestamp when customer opens)
- ‚úÖ Automated follow-up reminders (3 days, 7 days, before expiration)
- ‚úÖ Customer portal with all quotes/invoices
- ‚úÖ One-click resend
- ‚úÖ Mobile-friendly quote view

**NEW TABLE NEEDED:** `quote_deliveries`
```sql
CREATE TABLE quote_deliveries (
  id UUID PRIMARY KEY,
  work_order_id UUID REFERENCES work_orders(id),
  delivery_method TEXT, -- 'email', 'sms', 'portal', 'download'
  recipient_email TEXT,
  recipient_phone TEXT,
  sent_at TIMESTAMPTZ,
  viewed_at TIMESTAMPTZ,
  opened_at TIMESTAMPTZ,
  pdf_url TEXT,
  portal_link TEXT,
  delivery_status TEXT, -- 'sent', 'delivered', 'viewed', 'failed'
  error_message TEXT
);
```

---

### **Phase 4: Customer Response**
**Status:** `approved`, `rejected`, or `requires_changes`
**Tables:** `work_orders`, `quote_responses`, `signatures`, `work_order_status_history`

**Actions:**
- Customer accepts (with optional signature)
- Customer rejects (with reason)
- Customer requests changes
- Optional deposit payment

**Pain Points Found:**
- ‚ùå **No rejection tracking** - Don't know WHY customer rejected
- ‚ùå **No change request workflow** - Customer calls/emails changes
- ‚ùå **Deposit confusion** - Not clear how much deposit is required
- ‚ùå **No signature capture** - Have to print, sign, scan
- ‚ùå **Lost quotes** - Customer accepted verbally but no record

**Industry Best Practices:**
- ‚úÖ One-click accept/reject from portal
- ‚úÖ Required rejection reason (dropdown + notes)
- ‚úÖ Change request form (customer can specify what to change)
- ‚úÖ Digital signature capture (mobile-friendly)
- ‚úÖ Deposit payment at acceptance (optional)
- ‚úÖ Automated notification to contractor

**NEW TABLE NEEDED:** `quote_responses`
```sql
CREATE TABLE quote_responses (
  id UUID PRIMARY KEY,
  work_order_id UUID REFERENCES work_orders(id),
  response_type TEXT, -- 'accepted', 'rejected', 'changes_requested'
  rejection_reason TEXT,
  change_request_notes TEXT,
  signature_id UUID REFERENCES signatures(id),
  responded_at TIMESTAMPTZ,
  responded_by TEXT -- customer name/email
);
```

---

### **Phase 5: Change Orders**
**Status:** Still `approved` but with change order flag
**Tables:** `work_orders`, `change_orders`, `change_order_items`

**Actions:**
- Customer or contractor requests changes
- Create change order with new line items
- Customer approves change order
- Update total amount

**Pain Points Found:**
- ‚ùå **BIGGEST COMPLAINT** - Change orders are a nightmare
- ‚ùå **No change order workflow** - Have to create new quote
- ‚ùå **Disputes over scope** - Customer says "that was included"
- ‚ùå **Payment confusion** - Not clear what's owed for changes
- ‚ùå **No audit trail** - Can't prove what was agreed to

**Industry Best Practices:**
- ‚úÖ Separate change order entity (linked to original quote)
- ‚úÖ Change order approval workflow
- ‚úÖ Clear pricing for additions/deletions
- ‚úÖ Customer signature required
- ‚úÖ Audit trail of all changes
- ‚úÖ Original quote remains unchanged (for reference)

**NEW TABLE NEEDED:** `change_orders`
```sql
CREATE TABLE change_orders (
  id UUID PRIMARY KEY,
  work_order_id UUID REFERENCES work_orders(id),
  change_order_number TEXT UNIQUE,
  description TEXT,
  reason TEXT, -- 'customer_request', 'scope_change', 'unforeseen_work'
  status TEXT, -- 'draft', 'pending_approval', 'approved', 'rejected'
  subtotal DECIMAL,
  tax_amount DECIMAL,
  total_amount DECIMAL,
  requested_by UUID REFERENCES profiles(id),
  approved_by TEXT, -- customer name
  signature_id UUID REFERENCES signatures(id),
  created_at TIMESTAMPTZ,
  approved_at TIMESTAMPTZ
);

CREATE TABLE change_order_items (
  id UUID PRIMARY KEY,
  change_order_id UUID REFERENCES change_orders(id),
  item_type TEXT, -- 'addition', 'deletion', 'modification'
  description TEXT,
  quantity DECIMAL,
  unit_price DECIMAL,
  total DECIMAL
);
```

**Actions:**

**Pain Points Found:**
- ‚ùå **Double booking** - Techs get overbooked
- ‚ùå **No skill matching** - Wrong tech assigned to job
- ‚ùå **No customer confirmation** - Customer forgets appointment
- ‚ùå **No ETA updates** - Customer doesn't know when tech will arrive
- ‚ùå **No route optimization** - Techs waste time driving

**Industry Best Practices:**
- ‚úÖ Drag-and-drop calendar
- ‚úÖ Skill-based assignment
- ‚úÖ Automated customer confirmation (email + SMS)
- ‚úÖ Automated tech dispatch
- ‚úÖ Real-time ETA updates
- ‚úÖ Route optimization

- ‚úÖ Barcode/QR inventory scanning
